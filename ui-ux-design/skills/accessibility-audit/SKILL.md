# accessibility-audit

Audit UIs against WCAG 2.2. Auto-activates on a11y queries or via `/uix:accessibility-audit`. Bash is permitted only for contrast-ratio computation — no network, no filesystem writes.

## Triggers

"accessibility audit" · "WCAG check / audit / compliance" · "a11y review" · "is this accessible" · "check contrast" · "keyboard accessible" · "screen reader" · "target size" · "focus ring" · "ARIA" · "color contrast ratio" · "dragging alternative" · "accessible authentication".

## Process

1. **Ingest the target** (file, URL, screenshot, or description). If only a description is given, ask for code or a rendered example — an a11y audit without concrete markup is guesswork.

2. **Load WCAG criteria** from `${CLAUDE_PLUGIN_ROOT}/references/UI_MASTER_GUIDE.md` Part XII. Walk the four principles in order.

3. **For each criterion evaluate: Pass / Fail / N/A / Needs manual check.** Don't skip — explicit N/A is better than silent omission.

4. **Output format:**

   ### Summary
   Conformance estimate: [A / AA / AA-with-gaps / fails-AA]
   Critical blockers: N
   Warnings: N

   ### Perceivable

5. **For contrast checks**, if you have the actual color values, compute the WCAG ratio. Otherwise mark "Needs manual check" and list what to measure.

6. **For keyboard/focus checks** with code available, trace tab order in the markup. Flag any `tabindex="-1"` on interactive elements, missing focus styles (`outline: none` without replacement), and sticky headers that could obscure focus.

7. **For the new 2.2 criteria**, always evaluate explicitly even if unfamiliar — that's where most audits miss things.

## Ground rules

- AA is the baseline for production. AAA is a stretch goal — note but don't gate ship on AAA.
- Don't mark something "Pass" without evidence in the target; say "Likely pass, needs manual confirmation" if evidence is partial.
- Color-only indicators always fail WCAG 1.4.1 regardless of contrast.
- `outline: none` without a replacement focus style always fails 2.4.7.
- Missing `<label>` always fails 3.3.2 — `aria-label` on the input is acceptable only if the visible text matches.
- Don't trust automated checkers alone — always list what still needs manual verification.
